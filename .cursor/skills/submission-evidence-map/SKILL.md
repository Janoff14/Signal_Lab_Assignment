## When to Use

- User asks for submission readiness or "are we done?"
- README or checklist updates are requested.
- New stack or observability artifacts are added.
- Reviewer preparation before final push.

## Workflow
1. Read `SUBMISSION_CHECKLIST.md` and `RUBRIC.md`.
2. Map each requirement to exact repo paths.
3. Ensure reproducible commands are listed for run/verify/stop.
4. Flag missing proof and propose minimal artifacts to close gaps.
